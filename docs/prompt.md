# 記帳 Bot（Discord + LLM）目前規格與實作狀態

## 1) 專案目標

做一個可自然對話的 Discord 記帳機器人，並提供 Web Dashboard 進行管理、編輯、匯入匯出與設定。

---

## 2) 目前已完成（可用）

### 2.1 Discord Bot 核心

- Slash 指令可用（含初始化、預算、收支、查詢等）
- 訊息處理採 **LLM 優先**：先判斷意圖，再決定是否記帳/追問/引導設定
- Bot 處理訊息前會顯示「正在輸入」
- 記帳成功後依序發送：
  1. 本月預算使用量（Embed + 進度條）
  2. 記帳成功訊息（Embed）
  3. 閒聊回覆（LLM）

### 2.2 初始化流程（對話式）

- `/初始化` 若偵測到既有資料（交易、設定、釘選），會先跳確認（按鈕）
- 可確認清空後再開始初始化
- 初始化為多步驟對話，未完成前會持續引導
- 支援中途修正前面欄位（不是死流程）
- 目前主要欄位：
  - 每月預算
  - 每日提醒時間
  - 性別
  - 稱呼
  - 分帳本開關（先保留設定，不啟用帳本邏輯）
  - 分類設定

### 2.3 頻道命名同步

- 頻道名稱格式：`<稱呼>_<餘額數字>`
- 不含「餘額」兩字
- 支援負數
- 可由設定開關控制是否顯示餘額於頻道名稱

### 2.4 Web Dashboard（已可管理）

- 檢視最近交易、餘額、收入/支出統計
- 編輯單筆交易
- 刪除單筆交易
- 設定頁可展開/收合
- 設定項目：
  - 預算、提醒時間、性別、稱呼、貨幣
  - 分帳本開關（先保留）
  - 頻道名稱顯示餘額開關
  - 載具同步開關（先保留）
  - 分類（Tag 多選，可新增/刪除）
  - 帳本（Chip UI，可新增/刪除，先存設定）
  - 週期收支規則

### 2.5 週期收支（已「真正生效」）

- Bot 啟動後，每分鐘跑一次排程
- 會讀取各頻道的週期收支設定，自動入帳到 `transactions`
- 具去重機制（避免同一規則同一到期點重複入帳）
- 支援規則格式（每行一筆）：
  - `名稱,金額,每月1號 09:00,expense`
  - `名稱,金額,每週一 09:00,income`
  - `名稱,金額,每日 09:00,expense`

### 2.6 匯入 / 匯出（CSV / JSON）

- 匯出 API：
  - `GET /api/channel/:channelId/export?format=json`
  - `GET /api/channel/:channelId/export?format=csv`
- 匯入 API：
  - `POST /api/channel/:channelId/import`
  - `mode`: `append` 或 `replace`
  - `format`: `json` 或 `csv`
- Web 已有按鈕與檔案上傳流程

## 3) 目前技術實作（實際使用）

| 層面 | 技術 |
| --- | --- |
| Bot | `discord.js` (Node.js) |
| LLM | MiniMax（Anthropic 相容 API） |
| DB（Bot） | `sql.js`（記憶體載入 + 檔案同步機制） |
| DB（Web） | `better-sqlite3`（直接讀寫 SQLite） |
| Web 後端 | `express` |
| Web 前端 | 原生 HTML/CSS/JS |

補充：已處理 Bot / Web 同步讀寫同一 SQLite 檔案的一致性問題（Bot 查詢前會檢查並 reload 最新檔案）。

## 4) 預設分類（目前使用）

- 支出：餐飲、交通、購物、娛樂、房租/帳單、住宿、醫療、教育、投資、禮物、其他
- 收入：薪資、兼職、被動收入、紅包、禮物、生活費、其他

---

## 5) LLM 行為原則（已實作）

- 所有自然語言先走 LLM 判斷意圖
- 若 LLM 信心不足或失敗：
  - 只做 log / 引導澄清
  - **不自動寫入交易**
- 初始化期間，像「12000」這類輸入會優先當成初始化回答上下文，而不是直接記帳

---

## 6) 尚未完整完成（目前狀態）

- 載具同步：目前只有設定欄位與開關，尚未串接財政部 API
- 分帳本：目前只有 UI 與設定儲存，尚未分帳本記帳邏輯
- 圖表分析：尚未在 Web 端完成圖表與進階分析面板
- AI 直接查庫分析：尚未提供專門 query 介面

## 8) 測試建議（每次發版前）

- 初始化：有資料 / 無資料兩種路徑都測
- LLM 解析：正常句、模糊句、只有數字、混合語句
- Web 編輯後 Discord 查詢是否同步
- 週期收支是否準點且不重複入帳
- CSV/JSON 匯入匯出 round-trip（匯出再匯入）
