# 記帳 Bot（Discord + LLM）目前規格與進度（現況版）

## 1) 專案目標

建立一個可自然對話的 Discord 記帳機器人，搭配 Web Dashboard 完成日常管理（查詢、編輯、設定、匯入匯出）並保留擴充彈性。

---

## 2) 功能進度總覽

### 2.1 已完成（可直接使用）

#### Discord Bot

- 訊息處理採 LLM 優先（意圖判斷、記帳、查詢分析、追問）
- 記帳流程固定三段輸出：
  1. 本月預算使用量 Embed（含進度條）
  2. 記帳成功 Embed（含 Dashboard 小連結）
  3. LLM 簡短回饋
- Bot 處理訊息期間會顯示「正在輸入」
- 每頻道訊息佇列（避免連續訊息造成判斷衝突）
- 圖片記帳（MiniMax Vision）可嘗試解析收據/發票
- 查詢分析（LLM tool-plan）可處理比較、分類拆解、趨勢
- 單日詢問（今天/昨天開銷）會輸出摘要 Embed

#### Slash 指令（現行）

- `/初始化`
- `/初始化-共同記賬`
- `/記帳`
- `/收入`
- `/查詢`
- `/預算`（可設可查）
- `/儀表板`
- `/幫助`

#### 初始化與頻道規則

- `/初始化` 若偵測到舊資料會先顯示確認 Embed（按鈕）
- 初始化流程為對話式多步驟：預算、提醒時間、性別、稱呼
- 初始化未完成時會持續引導，且可修正前一題
- 未初始化頻道為「靜默模式」：一般對話不回覆
- 頻道資料完全以 `channel_id` 隔離（不同頻道互不影響）

#### 共同賬本（伺服器唯一）

- `/初始化-共同記賬` 可指定「伺服器唯一共同賬本頻道」
- 同伺服器若已存在共同賬本，切換時會先跳確認 Embed
- 確認切換後會清理舊共同賬本資料，再切到新頻道
- 在其他已初始化頻道可用自然語句轉入共同賬本，例如：
  - `給共同賬本添加 1000 元`
- 轉入時會雙邊入帳（來源頻道支出、共同賬本收入）

#### 頻道名稱同步

- 命名格式：`<稱呼>_<餘額數字>`
- 支援負數
- 是否顯示餘額可由設定控制
- 有排程同步器，避免 Bot/Web 混合修改後不同步

#### Web Dashboard

- 月分析（年/月下拉）
- 月收支統計與分類圓餅圖（收入/支出）
- 月條目列表雙模式：依日期 / 依分類
- 分類 badge 顏色區分
- 單筆交易編輯/刪除
- 設定面板可展開/收合
- 設定欄位：
  - 預算、提醒時間、性別、稱呼、貨幣
  - 分類 tags（可新增/刪除）
  - 對話風格 tags（可新增/刪除）
  - 週期收支（表單列引導，不需手打字串）
  - 頻道名稱顯示餘額開關
  - 載具同步開關（僅保留欄位）
- 前後端輸入驗證（預算/提醒時間/貨幣）與錯誤提示
- 匯出：JSON / CSV
- 匯入：JSON / CSV（append / replace）
- 手機版排版已做 RWD 修正

#### 排程功能

- 每分鐘檢查週期收支（含去重）
- 每分鐘檢查每日提醒（含當日去重）
- 每分鐘同步頻道名稱餘額

### 2.2 部分完成（可用但仍可加強）

- MiniMax Vision 解析準確率仍受圖片品質影響
- LLM query 分析已可用，但可再加更多「工具鏈」與穩定 fallback
- 共同賬本已可初始化與轉入，尚未有完整成員權限/審計機制

---

### 2.3 尚未完成 / 僅保留欄位

- 載具同步：尚未串接財政部 API
- 分帳本（split books / ledgers）：目前不啟用實際記帳邏輯
- 操作審計：尚未提供「誰改了哪筆資料」歷史

## 3) 技術實作（目前）

| 層面 | 技術 |
| --- | --- |
| Bot | `discord.js` (Node.js) |
| LLM | MiniMax（Anthropic 相容 API） |
| DB（Bot） | `sql.js`（記憶體 + 檔案 mtime 自動 reload） |
| DB（Web） | `better-sqlite3` |
| Web 後端 | `express` |
| Web 前端 | 原生 HTML / CSS / JS + Chart.js |

補充：Bot/Web 共享同一 SQLite，Bot 在查詢前會檢查檔案更新並 reload，以降低資料不一致風險。

## 4) 預設分類（現行）

- 支出：餐飲、交通、購物、娛樂、房租/帳單、住宿、日常生活、醫療、教育、投資、禮物、其他
- 收入：薪資、兼職、被動收入、紅包、生活費、其他

## 5) LLM 行為原則（現行）

- 所有自然語言先走 LLM 路由
- LLM 失敗或低信心時：
  - 只記 log / 引導澄清
  - 不自動寫入交易（避免誤記帳）
- 初始化模式下，輸入會優先視為初始化回答（例如單純 `12000`）
- 分類優先受 `allowedCategories` 約束，並有別名/回退正規化

## 7) 測試重點（每次發版前）

- 初始化：新頻道 / 舊資料重置 / 共同賬本切換
- LLM：一般記帳、模糊句、連續訊息、圖片記帳
- 共同賬本：跨頻道轉入雙邊入帳是否一致
- Web 編輯後 Bot 是否即時讀到最新資料
- 週期收支與每日提醒是否準點且不重複
- CSV/JSON 匯出再匯入（round-trip）
